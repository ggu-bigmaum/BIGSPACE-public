## #2. Vite 프로덕션 빌드 시 .env 미인식 (2026-04-02)

### 증상
- 서버 배포 후 **로그인 페이지 안 뜸** (빈 화면)
- 로그아웃 버튼도 보이지 않음
- 빌드 성공, PM2 정상, .env 정상인데 이전 버전처럼 보임

### 원인
- 브라우저 콘솔에서 `Firebase: Error (auth/invalid-api-key)` 발생
- Vite의 `root`가 `client/`로 설정되어 있어, **빌드 시 `client/.env`에서 환경변수를 찾음**
- 실제 `.env`는 프로젝트 루트(`/home/rocky/BIGSPACE-public/.env`)에 위치
- `npm run dev` (개발모드)에서는 상위 폴더까지 유연하게 탐색하여 문제 없었음
- `npm run build` (프로덕션)에서는 `root` 기준으로만 `.env`를 찾아 `VITE_FIREBASE_*` 변수가 빌드에 포함되지 않음

### 해결
`vite.config.ts`에 `envDir` 추가:
```typescript
envDir: path.resolve(import.meta.dirname),  // 프로젝트 루트에서 .env 읽기
```

### 커밋
- `0cce3be` — Fix: Vite envDir 설정 추가 — 프로덕션 빌드에서 루트 .env 인식

### 교훈
- **로컬 dev와 프로덕션 build의 동작 차이**를 인지해야 함
- 서버에서 안 될 때 **브라우저 Console(F12)부터 먼저 확인**할 것
- `VITE_` 접두사 변수는 빌드 시점에 코드에 박히므로, `.env` 수정 후 반드시 `npm run build` 필요

## #3. Firebase 구글 로그인 — unauthorized-domain (2026-04-02)

### 증상
- 로그인 페이지는 정상 표시됨
- "Google 계정으로 로그인" 클릭 시 **"Google 로그인 실패 — Firebase: Error (auth/unauthorized-domain)"** 에러

### 원인
- Firebase Authentication은 **허용된 도메인에서만** 로그인 요청을 받음
- 로컬(`localhost`)은 기본 허용이라 개발 시 문제 없었음
- 서버 IP(`168.107.19.25`)는 Firebase에 등록되지 않아 차단됨

### 해결
1. Firebase Console → bigspace-public 프로젝트
2. **Authentication** → **Settings** → **Authorized domains**
3. **Add domain** → `168.107.19.25` 추가

### 교훈
- Firebase 구글 로그인을 서버에 배포할 때 **도메인/IP 등록 필수**
- 나중에 도메인(예: bigspace.kr) 연결 시에도 Firebase에 해당 도메인 추가 필요

## #1. 서버 최초 배포 시 반복 발생 이슈 (2026-03-26)

### 1-1. 포트 충돌 (EADDRINUSE)

**증상**
- `pm2 restart` 후 앱이 뜨지 않음
- 에러 로그에 `Error: listen EADDRINUSE 0.0.0.0:5000`

**원인**
- `pm2 restart`를 반복하면 이전 프로세스가 완전히 종료되지 않고 포트를 점유한 채로 남음
- 특히 `pm2 kill` 없이 `pm2 start`를 여러 번 실행하면 프로세스가 증식함

**해결**
```bash
pm2 kill
sudo fuser -k 5000/tcp
cd ~/BIGSPACE-public
pm2 start npm --name bigspace -- run start
pm2 save
```

> `fuser` 앞에 반드시 `sudo` 붙여야 함. 없으면 권한 오류로 실패.

### 1-2. git clone 시 .env 삭제

**증상**
- `git clone` 후 앱 실행 시 `DATABASE_URL must be set` 에러
- 또는 `injecting env (0)` 로그와 함께 Firebase 키 미인식

**원인**
- `.env`는 `.gitignore`에 포함되어 있어 git에 저장되지 않음
- `clone`하면 폴더가 새로 생성되면서 기존 `.env` 사라짐

**해결**
- 서버 코드 업데이트는 **clone 대신 pull** 사용
- `.env`는 최초 1회만 서버에서 직접 생성하고 이후 건드리지 않음

```bash
cd ~/BIGSPACE-public
git checkout -- package-lock.json
git pull
npm install
npm run build
pm2 restart bigspace
```

### 1-3. npm run build 누락

**증상**
- `git pull` 후 코드가 반영되지 않음 (이전 버전처럼 보임)
- 변경사항이 분명히 push됐는데 서버에서 안 보임

**원인**
- 서버는 빌드된 `dist/index.cjs` 파일을 실행함
- `git pull`로 소스코드를 받아도 **빌드를 다시 해야** 반영됨
- 로컬 `npm run dev`는 자동으로 처리해주지만 서버는 수동 빌드 필수

**해결**
- 코드 업데이트 후 반드시 `npm run build` 실행

> ⚠️ **`npm run build`를 빠뜨리면 이전 코드가 계속 실행됩니다.**
